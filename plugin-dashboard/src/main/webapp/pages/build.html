In order to make the features available in your Hippo project, you have to rebuild it. You don't need to do this for every single feature you add to the project. Rebuilding once after you added a batch of features is sufficient.

Build instructions

You can build your project in the root folder of your project, using the following command:

mvn verify

To run the CMS, site and Essentials WARs, issue the following command from the project root directory:

mvn -Pcargo.run -Drepo.path=storage -Drepo.bootstrap=true

By default, the repository content is stored in the directory target/storage. This has the effect that when you run mvn clean, the repository content, along with all changes and additions you made during development, are erased.

To avoid accidents in this area, it is a best practice to specify a different location for storing the repository, outside the target directory.

This is expressed by the repo.path parameter in he command above.

In order to make sure that repository settings pertaining to new project dependencies are correctly bootstrapped into the repository, make sure you specify the repo.bootstrap parameter. This is especially important when restarting the project after adding plugins.
